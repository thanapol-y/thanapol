เว็บนี้จัดทำขึ้นเพื่ออธิบาย ขั้นตอนการทำงานของระบบฐานข้อมูล ระบบจัดการกิจกรรม


ส่วนนี้เป็นส่วนของตาราง faculty โดยประกอบไปด้วย Faculty_ID (เก็บรหัสของคณะ) เก็บด้วย varchar(20) และเป็น primary key
และ Faculty_Name (เก็บชื่อคณะ) เก็บด้วย varchar(100)



ส่วนตรงนี้เป็นส่วนของตาราง branch โดยประกอบไปด้วย Branch_ID (เก็บรหัสของสาขา) เก็บด้วย varchar(20) และเป็น primary key
และ Branch_Name (เก็บชื่อสาขา) เก็บด้วย varchar(100)



ส่วนตรงนี้เป็นส่วนของตาราง department โดยประกอบไปด้วย Department_ID (เก็บรหัสของแผนก) เก็บด้วย varchar(20) และเป็น primary key
และ Department_Name (เก็บชื่อสาขา) เก็บด้วย varchar(100)



ส่วนตรงนี้เป็นส่วนของตาราง activity_type โดยประกอบไปด้วย Activity_Type_ID (เก็บรหัสของประเภทกิจกรรม) เก็บด้วย varchar(20) และเป็น primary key
และ Activity_Type_Name (เก็บชื่อของประภทกิจกรรม) เก็บด้วย varchar(100)



ส่วนตรงนี้เป็นส่วนของตาราง student โดยประกอบไปด้วย Student_ID (เก็บรหัสของนักศึกษา) เก็บด้วย varchar(20) และเป็น primary key
Student_Name (เก็บชื่อของนักศึกษา) เก็บด้วย varchar(100)
Student_Email (เก็บอีเมลของนักศึกษา) เก็บด้วย varchar(100)
และ Student_Phone (เก็บเบอร์ของนักศึกษา) เก็บด้วย varchar(20)
โดยมี Faculty_ID และ Branch_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง faculty(เพื่อบอกว่านักศึกษาคนนั้นๆอยู่คณะอะไร) และ branch(เพื่อบอกว่านักศึกษาคนนั้นๆอยู่สาขาอะไร) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางfaculty,branch) หากยังมีข้อมูลในตารางลูก(ตารางstudent)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง dean โดยประกอบไปด้วย Dean_ID (เก็บรหัสของรองคณบดี) เก็บด้วย varchar(20) และเป็น primary key
Dean_Name (เก็บชื่อของรองคณบดี) เก็บด้วย varchar(100)
และ Dean_Email (เก็บอีเมลของรองคณบดี) เก็บด้วย varchar(100)
โดยมี Department_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง department(เพื่อบอกว่าคณบดีอยู่แผนกอะไร) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางdepartment) หากยังมีข้อมูลในตารางลูก(ตารางdean)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง activity_head โดยประกอบไปด้วย Activity_Head_ID (เก็บรหัสของหัวหน้ากิจกรรม) เก็บด้วย varchar(20) และเป็น primary key
Activity_Head_Name (เก็บชื่อของหัวหน้ากิจกรรม) เก็บด้วย varchar(100)
Activity_Head_Email (เก็บอีเมลของหัวหน้ากิจกรรม) เก็บด้วย varchar(100)
และ Activity_Head_Phone (เก็บเบอร์ของหัวหน้ากิจกรรม) เก็บด้วย varchar(20)
โดยมี Department_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง department(เพื่อบอกว่าหัวหน้ากิจกรรมอยู่แผนกอะไร) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางdepartment) หากยังมีข้อมูลในตารางลูก(ตารางactivity_head)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง club โดยประกอบไปด้วย Club_ID (เก็บรหัสของสโมสรนักศึกษา) เก็บด้วย varchar(20) และเป็น primary key
Club_Name (เก็บชื่อของสโมสรนักศึกษา) เก็บด้วย varchar(100)
และ Club_Email (เก็บอีเมลของสโมสรนักศึกษา) เก็บด้วย varchar(100)
โดยมี Faculty_ID และ Branch_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง faculty(เพื่อบอกว่าสโมสรนักศึกษาคนนั้นๆอยู่คณะอะไร) และ branch(เพื่อบอกว่าสโมสรนักศึกษาคนนั้นๆอยู่สาขาอะไร) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางfaculty,branch) หากยังมีข้อมูลในตารางลูก(ตารางclub)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง activity โดยประกอบไปด้วย Activity_ID (เก็บรหัสของกิจกรรม) เก็บด้วย varchar(20) และเป็น primary key
Activity_Name (เก็บชื่อของกิจกรรม) เก็บด้วย varchar(100)
Activity_Details (เก็บรายละเอียดของกิจกรรม) เก็บด้วย text
Academic_Year (เก็บปีการศึกษา) เก็บด้วย int(4)
Activity_Date (เก็บวันที่จัดกิจกรรม) เก็บด้วย data
Activity_Time (เก็บเวลาจัดกิจกรรม) เก็บด้วย time
Activity_Location (เก็บสถนาที่จัดกิจกรรม) เก็บด้วย varchar(100)
Maximum_Capacity (เก็บจำนวนผู้เข้าร่วมกิจกรรม) เก็บด้วย int(6)
และ Deadline (เก็บวันเวลาสุดท้ายที่เปิดรับลงทะเบียน) เก็บด้วย datetime
โดยมี Activity_Type_ID , Dean_ID และ Activity_Head_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง activity_type(เพื่อแยกประเภทกกิจกรรม), dean(เพื่อบอกว่าใครเป็นคนอนุมัติกิจกรรม) และ Activity_Head_ID(เพื่อบอกว่าใครเป็นหัวหน้ากิจกรรมนั้นๆ) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางactivity_type,dean,activity_head) หากยังมีข้อมูลในตารางลูก(ตารางactivity)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง registration โดยประกอบไปด้วย Registration_Date (เก็บวันเวลาที่กดลงทะเบียน) เก็บด้วย datetime
และ QR_Code_Stat (เก็บเลขqrcodeของการลงทะเบียน) เก็บด้วย varchar(100)
โดยมี Student_ID และ Activity_ID เป็นทั้ง primary key และ foreign key โดยทำหน้าที่เป็นตัวเชื่อมว่า นักศึกษาคนไหน ลงทะเบียนกิจกรรมนั้นๆไป โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง student และ activity โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางstudent,activity) หากยังมีข้อมูลในตารางลูก(ตารางregistration)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข



ส่วนตรงนี้เป็นส่วนของตาราง check_in โดยประกอบไปด้วย CheckIn_Time (เก็บวันเวลาที่เข้าร่วมกิจกรรม) เก็บด้วย datetime
โดยมี Student_ID และ Activity_ID เป็นทั้ง primary key โดยทำหน้าที่เป็นตัวเชื่อมว่า นักศึกษาคนไหน ลงทะเบียนกิจกรรมนั้นๆไป โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง student และ activity โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางstudent,activity) หากยังมีข้อมูลในตารางลูก(ตารางcheck_in)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข
และมี Club_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้ อ้างอิงจากตาราง club(เพื่อบอกได้ว่าสโมสรคนไหนสแกนให้) โดยบอกว่าห้ามลบข้อมูลในตารางแม่(ตารางclub) หากยังมีข้อมูลในตารางลูก(ตารางcheck_in)อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข
