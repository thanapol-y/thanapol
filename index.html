เว็บนี้จัดทำขึ้นเพื่ออธิบาย ขั้นตอนการทำงานของระบบฐานข้อมูล ระบบจัดการกิจกรรม

ส่วนนี้เป็นส่วนของตาราง faculty โดยประกอบไปด้วย

  • Faculty_ID (เก็บรหัสของคณะ) เก็บด้วย varchar(20) และเป็น primary key
  • Faculty_Name (เก็บชื่อคณะ) เก็บด้วย varchar(100)

ส่วนนี้เป็นส่วนของตาราง branch โดยประกอบไปด้วย

  • Branch_Name (เก็บชื่อสาขา) เก็บด้วย varchar(100)

ส่วนนี้เป็นส่วนของตาราง activity_type โดยประกอบไปด้วย

ส่วนนี้เป็นส่วนของตาราง student โดยประกอบไปด้วย

  • Student_Name (เก็บชื่อของนักศึกษา) เก็บด้วย varchar(100)
  • Student_Email (เก็บอีเมลของนักศึกษา) เก็บด้วย varchar(100)
  • Student_Phone (เก็บเบอร์ของนักศึกษา) เก็บด้วย varchar(20)
Foreign Key Constraint: มี Faculty_ID และ Branch_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้อ้างอิงจากตาราง faculty (เพื่อบอกว่านักศึกษาคนนั้นๆอยู่คณะอะไร) และ branch (เพื่อบอกว่านักศึกษาคนนั้นๆอยู่สาขาอะไร) โดยบอกว่าห้ามลบข้อมูลในตารางแม่ (ตาราง faculty, branch) หากยังมีข้อมูลในตารางลูก (ตาราง student) อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข

ส่วนนี้เป็นส่วนของตาราง activity โดยประกอบไปด้วย

  • Activity_ID (เก็บรหัสของกิจกรรม) เก็บด้วย varchar(20) และเป็น primary key
  • Activity_Name (เก็บชื่อของกิจกรรม) เก็บด้วย varchar(100)
  • Activity_Details (เก็บรายละเอียดของกิจกรรม) เก็บด้วย text
  • Academic_Year (เก็บปีการศึกษา) เก็บด้วย int(4)
  • Activity_Date (เก็บวันที่จัดกิจกรรม) เก็บด้วย date
  • Activity_Time (เก็บเวลาจัดกิจกรรม) เก็บด้วย time
  • Activity_Location (เก็บสถานที่จัดกิจกรรม) เก็บด้วย varchar(100)
  • Maximum_Capacity (เก็บจำนวนผู้เข้าร่วมกิจกรรม) เก็บด้วย int(6)
  • Deadline (เก็บวันเวลาสุดท้ายที่เปิดรับลงทะเบียน) เก็บด้วย datetime
Foreign Key Constraint: มี Activity_Type_ID, Dean_ID และ Activity_Head_ID เป็น foreign key โดยที่จะทำเป็น Referential Constraint โดยสั่งให้อ้างอิงจากตาราง activity_type (เพื่อแยกประเภทกิจกรรม), dean (เพื่อบอกว่าใครเป็นคนอนุมัติกิจกรรม) และ activity_head (เพื่อบอกว่าใครเป็นหัวหน้ากิจกรรมนั้นๆ) โดยบอกว่าห้ามลบข้อมูลในตารางแม่ (ตาราง activity_type, dean, activity_head) หากยังมีข้อมูลในตารางลูก (ตาราง activity) อยู่ และให้อัพเดตข้อมูลในตารางลูกด้วย หากตารางแม่มีการแก้ไข

  • QR_Code_Stat (เก็บเลข qrcode ของการลงทะเบียน) เก็บด้วย varchar(100)
